Выполнил

Участник программы профессиональной переподготовки по профессии "web-разработчик" -
Королев Дмитрий

Тема урока

Адаптивный web-сайт

Решение

1.1 Создан новый проект с именем adaptive-website. Внутри проекта созданы файлы index.html и styles.css, папка images для хранения изображений и папка fonts для хранения шрифтов..
1.2 В index.html создана базовая структуру HTML-документа.

2.1 В styles.css стилизованя элементы страницы и подключены шрифты из папки fonts. С сайта Google Fonts скачан и добавлен на страницу шрифт "Open Sans"
2.2 Подобраны и добавлены изображения, которые адаптируются под размеры экрана. Установлены в качестве фоновой картинки тела страницы и адаптируются в css с помощью аналога srcset, функции image-set().
2.3 Добавлены медиа-запросы для изменения макета на разных устройствах. Создан адаптивный макет, который корректно отображается на мобильных устройствах и планшетах

3.1 В DevTools браузера страница протестирована на разных устройствах, проверена корректность загрузки шрифтов и изображений. Исправлены ошибки некорректного отображения страницы на каком-либо устройстве.
3.2 В шапке страницы с помощью z-index меню навигации накладывается на логотип при уменьшении размеров окна браузера.

4.1 Создан репозиторий на GitHub, инициализирован локальный репозиторий и связан с удалённым. Настроена публикация с ветки main.
4.2 Внесено изменение в проект, опубликовано, проверено, что изменения отобразились корректно.

5.1 Используя онлайн-генератор создан и установлен собственный favicon, отображается корректно.
5.2 Добавлены элементы, которые демонстрируют центрирование контента и работу с переполнением.
5.3 Отчет о кроссбраузерном тестировании:
     Первая трудность которая возникла, это использование srcset для вонового изображения. Так и не разобрался как его применить, поэтому воспользовался аналогичной функцией css.
     При использовании элементов, со строго заданной шириной, превышающих ширину экранов мобильных устройств, появляется полоса прокрутки, сдвигающая всю структуру страницы влево, включая шапку и подвал. Уменьшал ширину элемента, заведомо меньше ширины экранов мобильных устройств.
     Для декстопного варианта страницы, решил закрепить шапку сайта, для удобной навигации по якорным ссылкам, при этом все ссылки съехали вверх на высоту шапки. Пришлось у якорных элементов, устанавливать падинг на высоту шапки и одновременно отрицательный маржин на эту же величину.
